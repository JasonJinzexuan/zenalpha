### 2.4 L3 — 线段构建

| 测试 | 线段数 | 终结类型 | 评估 |

| Test 1 | 1 segment | FIRST_KIND | ✅ 特征序列顶分型，无缺口 |

| Test 3 | 0 segments | N/A | ✅ 只有1笔无法形成线段 |

**结论:** L3 线段构建正确。特征序列分析、包含处理、分型检测均按缠论规则工作。
**修复记录:**
- 修复了 `_try_terminate()` 只检查单方向的问题，增加了双向检测
- 测试数据需要确保特征序列元素间无包含关系（否则合并后不足3个元素）

### 2.5 L4 — 中枢

| 测试 | 中枢数 | 评估 |
|------|--------|------|
| 所有测试 | 0 | ⚠️ 需要至少3个方向交替的线段重叠区域才能形成中枢 |

**说明:** 测试数据不够长/复杂，未能形成3段以上重叠区间。需要 100+ 根K线且包含明显的区间重叠才能触发中枢。这是**数据不足**的问题，非代码问题。

### 2.6 L5-L7 — 走势/背驰/买卖点

| 层级 | 结果 | 评估 |
|------|------|------|
| L5 走势 | 无 | ⚠️ 需要中枢后才能判断走势类型 |
| L6 背驰 | 0 divergences | ⚠️ 需要同级别的走势完成后比较MACD面积 |
| L7 买卖点 | 0 signals | ⚠️ 需要背驰判断后才能生成信号 |

**说明:** L4-L7 形成依赖链 — 没有中枢就没有走势，没有走势就没有背驰，没有背驰就没有买卖点。这是**缠论理论的正常逻辑**，不是 bug。真实市场数据（200-500根K线）应能触发完整链路。

### 2.7 L8 — 区间套

未触发。区间套需要多级别数据同时分析（如日线 + 30分钟线），当前测试只用了单级别数据。

## 5. 数据连通性评估

### 当前架构

```
外部调用方 → POST /analyze {klines: [...]} → Pipeline → 结果
                                                ↑
                                         数据由调用方提供
```

### 缺失的数据链路

| 组件 | 状态 | 说明 |
|------|------|------|
| `PolygonClient` | ✅ 已实现 | 支持多级别K线获取 |
| `DataSource` Protocol | ✅ 已定义 | 抽象接口 |
| `CsvLoader` | ✅ 已实现 | 本地CSV回测 |
| `/scan` endpoint (自动拉数据) | ❌ 缺失 | 需要新增 |
| `POLYGON_API_KEY` 注入 | ❌ 缺失 | 需要加到K8s Secret |
| 定时扫描 CronJob | ❌ 缺失 | 需要新增 |

### 需要补充

1. **`/scan` endpoint**: 接收 `{instruments: ["AAPL", "TSLA"], timeframe: "1d"}` → 内部调用 PolygonClient 获取数据 → 跑 Pipeline → 返回信号
2. **K8s Secret**: 存储 POLYGON_API_KEY，mount 到 agent-service pod
3. **CronJob**: 每日收盘后自动扫描股票池
